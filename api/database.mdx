The Database API allows you to manage database connections, browse schemas and tables, and access metadata. This API includes 25 endpoints for comprehensive database management.

## List databases

Get a list of all databases the current user has access to.

```bash
GET /api/database
```

<CodeGroup>
```bash curl
curl -X GET \
  https://your-metabase.com/api/database \
  -H 'X-Metabase-Session: SESSION_TOKEN'
```

```javascript JavaScript
const databases = await fetch('https://your-metabase.com/api/database', {
  headers: { 'X-Metabase-Session': sessionToken }
}).then(r => r.json());
```

```python Python
response = requests.get(
    'https://your-metabase.com/api/database',
    headers={'X-Metabase-Session': session_token}
)
databases = response.json()
```
</CodeGroup>

### Response

Returns an array of database objects with connection details and metadata.

```json
{
  "data": [
    {
      "id": 1,
      "name": "Sample Database",
      "engine": "postgres",
      "created_at": "2024-01-15T10:00:00Z",
      "is_sample": false,
      "features": ["basic-aggregations", "nested-queries"]
    }
  ]
}
```

## Create database

Add a new database connection.

```bash
POST /api/database
```

<CodeGroup>
```bash curl
curl -X POST \
  https://your-metabase.com/api/database \
  -H 'Content-Type: application/json' \
  -H 'X-Metabase-Session: SESSION_TOKEN' \
  -d '{
    "name": "Production DB",
    "engine": "postgres",
    "details": {
      "host": "localhost",
      "port": 5432,
      "dbname": "production",
      "user": "metabase",
      "password": "secret"
    }
  }'
```

```javascript JavaScript
const database = await fetch('https://your-metabase.com/api/database', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-Metabase-Session': sessionToken
  },
  body: JSON.stringify({
    name: 'Production DB',
    engine: 'postgres',
    details: {
      host: 'localhost',
      port: 5432,
      dbname: 'production',
      user: 'metabase',
      password: 'secret'
    }
  })
}).then(r => r.json());
```
</CodeGroup>

### Request body

<ParamField body="name" type="string" required>
  Display name for the database
</ParamField>

<ParamField body="engine" type="string" required>
  Database engine type (postgres, mysql, h2, mongo, etc.)
</ParamField>

<ParamField body="details" type="object" required>
  Connection details specific to the database engine
  
  Common fields:
  - `host` - Database host
  - `port` - Database port
  - `dbname` - Database name
  - `user` - Username
  - `password` - Password
</ParamField>

<ParamField body="is_full_sync" type="boolean">
  Whether to perform a full sync (default: true)
</ParamField>

<ParamField body="schedules" type="object">
  Sync and cache field values schedules
</ParamField>

## Update database

Update an existing database connection.

```bash
PUT /api/database/{id}
```

<CodeGroup>
```bash curl
curl -X PUT \
  https://your-metabase.com/api/database/1 \
  -H 'Content-Type: application/json' \
  -H 'X-Metabase-Session: SESSION_TOKEN' \
  -d '{
    "name": "Updated Database Name",
    "details": {
      "host": "new-host.example.com"
    }
  }'
```
</CodeGroup>

### Parameters

<ParamField path="id" type="integer" required>
  Database ID
</ParamField>

### Request body

All fields are optional. Only include fields you want to update.

<ParamField body="name" type="string">
  Database display name
</ParamField>

<ParamField body="details" type="object">
  Connection details to update
</ParamField>

<ParamField body="is_full_sync" type="boolean">
  Enable or disable full sync
</ParamField>

## Delete database

Permanently delete a database connection.

```bash
DELETE /api/database/{id}
```

<CodeGroup>
```bash curl
curl -X DELETE \
  https://your-metabase.com/api/database/1 \
  -H 'X-Metabase-Session: SESSION_TOKEN'
```
</CodeGroup>

<Warning>
Deleting a database will also delete all questions, dashboards, and other content that depends on it. This action cannot be undone.
</Warning>

## Get autocomplete suggestions

Get SQL autocomplete suggestions for native queries.

```bash
GET /api/database/{id}/autocomplete_suggestions
```

<CodeGroup>
```bash curl
curl -X GET \
  'https://your-metabase.com/api/database/1/autocomplete_suggestions?query=SELECT%20*%20FROM%20' \
  -H 'X-Metabase-Session: SESSION_TOKEN'
```
</CodeGroup>

### Parameters

<ParamField path="id" type="integer" required>
  Database ID
</ParamField>

<ParamField query="query" type="string" required>
  Partial SQL query to get suggestions for
</ParamField>

### Response

Returns an array of autocomplete suggestions including table names, column names, and SQL keywords.

Common error codes for database endpoints:

<ResponseField name="400 Bad Request" type="error">
  Invalid database configuration or connection details
</ResponseField>

<ResponseField name="403 Forbidden" type="error">

</ResponseField>

</ResponseField>
