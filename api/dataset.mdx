title: Dataset
description: Execute queries and retrieve data programmatically through the Metabase API
---

The Dataset API allows you to execute ad-hoc queries and retrieve data. This is the primary endpoint for running queries programmatically. The API includes 8 endpoints.

## Execute query

Execute an ad-hoc query and retrieve results.

```bash
POST /api/dataset
```

<CodeGroup>
```bash curl
curl -X POST \
  https://your-metabase.com/api/dataset \
  -H 'Content-Type: application/json' \
  -H 'X-Metabase-Session: SESSION_TOKEN' \
  -d '{
    "database": 1,
    "type": "query",
    "query": {
      "source-table": 5,
      "limit": 10
    }
  }'
```

```javascript JavaScript
const results = await fetch('https://your-metabase.com/api/dataset', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-Metabase-Session': sessionToken
  },
  body: JSON.stringify({
    database: 1,
    type: 'query',
    query: {
      'source-table': 5,
      limit: 10
    }
  })
}).then(r => r.json());
```

```python Python
response = requests.post(
    'https://your-metabase.com/api/dataset',
    headers={'X-Metabase-Session': session_token},
    json={
        'database': 1,
        'type': 'query',
        'query': {
            'source-table': 5,
            'limit': 10
        }
    }
)
results = response.json()
```
</CodeGroup>

### Request body

<ParamField body="database" type="integer">
  Database ID to query (optional for saved question queries)
</ParamField>

<ParamField body="type" type="string" required>
  Query type: "query" (MBQL) or "native" (SQL)
</ParamField>

<ParamField body="query" type="object" required>
  MBQL query object
  
  Example MBQL query:
  ```json
  {
    "source-table": 5,
    "aggregation": [["count"]],
    "breakout": [["field", 12, null]],
    "limit": 100
  }
  ```
</ParamField>

<ParamField body="native" type="object">
  Native SQL query (when type is "native")
  
  Example:
  ```json
  {
    "query": "SELECT * FROM orders LIMIT 100",
    "template-tags": {}
  }
  ```
</ParamField>

<ParamField body="parameters" type="array">
  Parameter values for template tags
</ParamField>

<ParamField body="constraints" type="object">
  Query constraints like max results
</ParamField>

### Response

<ResponseField name="data" type="object">
  Query results
  
  - `rows` - Array of result rows
  - `cols` - Array of column metadata  
  - `rows_truncated` - Number of rows truncated (if limited)
</ResponseField>

<ResponseField name="database_id" type="integer">
  Database that was queried
</ResponseField>

<ResponseField name="status" type="string">
  Query status: "completed", "failed", etc.
</ResponseField>

<ResponseField name="row_count" type="integer">
  Number of rows returned
</ResponseField>

<ResponseField name="running_time" type="integer">
  Query execution time in milliseconds
</ResponseField>

<ResponseField name="json_query" type="object">
  The query that was executed
</ResponseField>

```json
{
  "data": {
    "rows": [
      [1, "Product A", 29.99],
      [2, "Product B", 49.99]
    ],
    "cols": [
      {
        "name": "ID",
        "base_type": "type/Integer",
        "display_name": "ID"
      },
      {
        "name": "NAME",
        "base_type": "type/Text",
        "display_name": "Name"
      },
      {
        "name": "PRICE",
        "base_type": "type/Float",
        "display_name": "Price"
      }
    ]
  },
  "status": "completed",
  "row_count": 2,
  "running_time": 145
}
```

<Note>
This endpoint does not use query caching. Results are always fresh from the database.
</Note>

## Export query results

Execute a query and download results in a specific format.

```bash
POST /api/dataset/{export-format}
```

<CodeGroup>
```bash curl
curl -X POST \
  https://your-metabase.com/api/dataset/csv \
  -H 'Content-Type: application/json' \
  -H 'X-Metabase-Session: SESSION_TOKEN' \
  -d '{
    "query": {
      "database": 1,
      "type": "query",
      "query": {
        "source-table": 5
      }
    },
    "visualization_settings": {},
    "format_rows": true,
    "pivot_results": false
  }' \
  -o results.csv
```

```javascript JavaScript
const response = await fetch('https://your-metabase.com/api/dataset/xlsx', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-Metabase-Session': sessionToken
  },
  body: JSON.stringify({
    query: {
      database: 1,
      type: 'query',
      query: { 'source-table': 5 }
    },
    visualization_settings: {},
    format_rows: true,
    pivot_results: false
  })
});

const blob = await response.blob();
```
</CodeGroup>

### Parameters

<ParamField path="export-format" type="string" required>
  Export format: csv, xlsx, or json
</ParamField>

### Request body

<ParamField body="query" type="object" required>
  Query object with database, type, and query/native fields
</ParamField>

<ParamField body="visualization_settings" type="object" required>
  Visualization settings (can be empty object)
</ParamField>

<ParamField body="format_rows" type="boolean">
  Apply cell formatting to values (default: false)
</ParamField>

<ParamField body="pivot_results" type="boolean">
  Pivot the results (default: false)
</ParamField>

### Response

Returns file data with appropriate content-type:
- CSV: `text/csv`
- XLSX: `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- JSON: `application/json`

## Get query metadata

Get metadata needed to construct a query.

```bash
POST /api/dataset/query_metadata
```

<CodeGroup>
```bash curl
curl -X POST \
  https://your-metabase.com/api/dataset/query_metadata \
  -H 'Content-Type: application/json' \
  -H 'X-Metabase-Session: SESSION_TOKEN' \
  -d '{
    "database": 1
  }'
```
</CodeGroup>

### Request body

<ParamField body="database" type="integer" required>
  Database ID to get metadata for
</ParamField>

### Response

Returns database metadata including tables, fields, and foreign key relationships needed for building queries.

## Convert to native query

Convert an MBQL query to native SQL.

```bash
POST /api/dataset/native
```

<CodeGroup>
```bash curl
curl -X POST \
  https://your-metabase.com/api/dataset/native \
  -H 'Content-Type: application/json' \
  -H 'X-Metabase-Session: SESSION_TOKEN' \
  -d '{
    "database": 1,
    "type": "query",
    "query": {
      "source-table": 5,
      "limit": 10
    },
    "pretty": true
  }'
```
</CodeGroup>

### Request body

<ParamField body="database" type="integer" required>
  Database ID (minimum: 1)
</ParamField>

<ParamField body="type" type="string" required>
  Query type ("query")
</ParamField>

<ParamField body="query" type="object" required>
  MBQL query to convert
</ParamField>

<ParamField body="pretty" type="boolean">
  Format the SQL with indentation (default: true)
</ParamField>

### Response

<ResponseField name="query" type="string">
  Native SQL query
</ResponseField>

```json
{
  "query": "SELECT * FROM orders LIMIT 10"
}
```

## Get parameter values

Get possible values for a query parameter.

```bash
POST /api/dataset/parameter/values
```

<CodeGroup>
```bash curl
curl -X POST \
  https://your-metabase.com/api/dataset/parameter/values \
  -H 'Content-Type: application/json' \
  -H 'X-Metabase-Session: SESSION_TOKEN' \
  -d '{
    "parameter": {
      "id": "param1",
      "type": "category",
      "target": ["dimension", ["field", 5, null]]
    },
    "field_ids": [5]
  }'
```
</CodeGroup>

### Request body

<ParamField body="parameter" type="object" required>
  Parameter definition with id, type, and target
</ParamField>

<ParamField body="field_ids" type="array">
  Field IDs to get values from
</ParamField>

### Response

Returns an array of possible parameter values.

## Generate pivot table

Generate a pivoted dataset for a query.

```bash
POST /api/dataset/pivot
```

<CodeGroup>
```bash curl
curl -X POST \
  https://your-metabase.com/api/dataset/pivot \
  -H 'Content-Type: application/json' \
  -H 'X-Metabase-Session: SESSION_TOKEN' \
  -d '{
    "database": 1,
    "type": "query",
    "query": {
      "source-table": 5,
      "aggregation": [["count"]],
      "breakout": [["field", 12, null], ["field", 13, null]]
    }
  }'
```
</CodeGroup>

### Request body

<ParamField body="database" type="integer" required>
  Database ID (minimum: 1)
</ParamField>

<ParamField body="type" type="string" required>
  Query type
</ParamField>

<ParamField body="query" type="object" required>
  Query with breakout fields for pivoting
</ParamField>

### Response

Returns pivoted data with row and column groupings.

<Tip>
Use pivot queries for creating cross-tab reports and matrices.
</Tip>
